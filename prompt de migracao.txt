Regras imutáveis durante toda a migração (copie e cole isso no seu raciocínio):
- Fidelidade absoluta: NÃO adicionar features, fluxos ou pacotes novos sem aprovação explícita.
- Manter exatamente a mesma lógica de negócio, autenticação, endpoints, schemas.
- Tipagem RPC end-to-end: exportar AppType do src/backend/app.ts e usar hc<AppType> no frontend.
- Hono sempre tipado: new Hono<{ Bindings: ...; Variables: ... }>()
- Schemas Zod: mover para src/shared/schemas/ e usar em ambos os lados (zValidator + infer).
- Zero hardcodes: strings via Paraglide, cores/tokens via DESIGN-SYSTEM, ícones via src/shared/icons.ts.
- Testes: pelo menos 1 teste vitest por endpoint crítico movido (happy path + erro).
- Integração: ZERO HTTP real no dev/prod → usar hooks.server.ts + app.fetch(request).
- Deploy: único wrangler deploy (sem turbo filters separados).

Passos obrigatórios da migração (faça um de cada vez, peça confirmação antes de avançar para o próximo):

1. Criar novo branch: git checkout -b refactor-single-app
2. Criar novo projeto SvelteKit na raiz (se necessário, mas prefira mover ficheiros existentes):
   pnpm create svelte@latest . -- com adapter-cloudflare, TypeScript, etc.
3. Mover e renomear pastas:
   - apps/web/src → src/frontend/
   - apps/api/src → src/backend/
   - packages/shared → src/shared/
4. Criar src/backend/app.ts com Hono principal (export type AppType = typeof app)
5. Criar hooks.server.ts em raiz/src/ com delegação para backend:
   import app from './backend/app';
   export const handle = async ({ event }) => {
     if (event.url.pathname.startsWith('/api')) {
       return app.fetch(event.request, event.platform?.env, event);
     }
     return resolve(event);
   };
6. Fundir configs: wrangler.toml único, vite.config.ts, svelte.config.js, tsconfig, vitest.config
7. Atualizar imports no frontend para hc<AppType> (criar src/frontend/lib/api/index.ts)
8. Mover schemas Zod para src/shared/schemas/ e ajustar imports
9. Ajustar bindings (D1, KV, R2) no wrangler.toml único + types.ts
10. Executar build, lint, test, dev (pnpm dev deve rodar frontend + backend integrado)
11. Testar manualmente: login, endpoints principais, hot reload
